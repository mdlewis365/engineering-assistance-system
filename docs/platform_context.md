# Platform Context

## Carter Relationship

EAS runs inside the private Carter/Synthetic OS application environment. Carter provides authenticated access, asynchronous job execution, governed generation, result streaming, and selected persistence.

## EAS Boundary

EAS is the engineering workflow. It is separate from the general Carter console and from SIS, although all are part of the same private environment.

## Supporting Documents

The implemented workflow accepts supporting documents as engineering evidence. Public documentation describes this conceptually and does not publish raw uploads or document parsing internals.

## What Is Not Included

This file does not disclose application routes, access-control implementation, private file handling, storage paths, model providers, prompt content, logs, or deployment details.
