# Engineering Modes

## Solve the Problem

Used when the user needs a concrete selected answer, sizing result, configuration, or action recommendation. Deterministic computation is used when the problem can be reduced to bounded equations, constraints, or candidate selection.

## Diagnose Root Cause

Used when the user needs to evaluate observed symptoms against possible causes. Deterministic checks may identify, eliminate, or leave unresolved candidate causes. Failed observed-condition criteria may be diagnostic evidence rather than a top-level design failure.

## Review Design

Used when the user asks whether an existing design, sizing, release package, or installation is acceptable. Deterministic checks can evaluate criteria, margins, and pass/fail conditions when inputs are sufficient.

## Suggest Improvements

Used when the user wants practical improvements to an existing system. Deterministic computation can screen candidate improvements where bounded metrics or constraints exist.

## Explore Novel Solution

Used when the user wants distinct new solution concepts. Deterministic computation can screen concepts at a bounded feasibility level, but output remains conceptual and review-gated.

## What Is Not Included

This document does not disclose private mode prompts, exact scoring logic, prompt templates, or routing internals.
