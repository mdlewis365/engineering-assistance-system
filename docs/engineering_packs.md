# Engineering Packs

## Purpose

Engineering packs are thin guidance layers for EAS. They help align source handling, mode behavior, domain language, MCM-safe planning, and report discipline.

They are not answer banks, hardcoded test fixtures, or public formula libraries.

## Public Pack Types

- Core guidance: source handling, deterministic computation discipline, evidence basis, and reporting constraints.
- Mode guidance: solve-problem, diagnose-root-cause, review-design, suggest-improvements, and explore-novel-solution behavior.
- Domain guidance: public-safe engineering domain framing where the current implementation has reusable pack coverage.

## Current Public Domain Coverage

The current implementation includes reusable guidance around domains such as:

- industrial exhaust airflow
- compressed air and pneumatics
- DC controls power
- fluid pump loops and hydraulic-style calculations
- mechanical power transmission
- structural mechanical brackets
- thermal enclosure cooling
- aerospace preliminary design and aerodynamics
- software systems engineering

## Selection Behavior

EAS uses the user problem, selected mode, supporting-file signals, and structured planning evidence to select or refine guidance. Public documentation does not disclose exact routing rules or keyword tables.

## What Is Not Included

This document does not include private pack text, exact pack IDs as routing contracts, keyword lists, prompt insertion details, formulas, schemas, or cloneable selection logic.
